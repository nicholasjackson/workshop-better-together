# 🏴‍☠️ Intermediate Puzzles

### 1. Terraform Modules
**Puzzle**: “Aye, modular code shares the spoils. How do ye group related resources into reusable treasure chests?”  
**Hint**: Break large configs into smaller, shareable parts.

---

### 2. Terraform State File
**Puzzle**: “Ye store the map of all created resources in a hidden ledger. Which file holds these secrets?”  
**Hint**: Tracks the current infrastructure and resource metadata.

---

### 3. Vault AppRole
**Puzzle**: “Machines too must prove their worth for secret plunder. Which Vault method grants them this right?”  
**Hint**: Allows applications to authenticate without user interaction.

---

### 4. Vault Dynamic Secrets
**Puzzle**: “To grant a service fleeting DB credentials, how might Vault mint them just in time?”  
**Hint**: On-demand credentials that expire automatically.

---

### 5. Vault PKI
**Puzzle**: “Arr! Certificates must be short-lived and minted fresh. Which Vault secrets engine conjures them on the fly?”  
**Hint**: A specialized engine that issues SSL certificates dynamically.

---

### 6. Packer Provisioners
**Puzzle**: “Beyond building the hull, ye must furnish the ship. Which step in Packer sets up and configures the OS or software?”  
**Hint**: Runs scripts or tools during image creation.

---

### 7. Packer Builders
**Puzzle**: “Each ship is forged in a distinct harbor—how does Packer define the place (AWS, Azure, local) to craft an image?”  
**Hint**: The section specifying where and how the machine image is built.

---

### 8. Packer Post-Processors
**Puzzle**: “After forging the vessel, some final touches be needed—like compressing or uploading. Which part of Packer does that?”  
**Hint**: A stage that runs last to finalize or distribute the image.

---

### 9. Ansible Roles
**Puzzle**: “Organize tasks by function, so ye may reuse them across your entire fleet. What is this chunk of Ansible called?”  
**Hint**: Bundles tasks, handlers, and variables into a reusable unit.

---

### 10. Ansible Handlers
**Puzzle**: “When a change is made, triggers must fire. Which mechanism in Ansible awaits a notify for actions like reloading a service?”  
**Hint**: A special task that executes only when invoked.

---

### 11. Ansible Vault
**Puzzle**: “Secrets in the open sea be dangerous. How do ye encrypt them in a YAML file?”  
**Hint**: Native Ansible feature for securing sensitive data in playbooks.

---

### 12. ansible-pull
**Puzzle**: “In this strange land, the host fetches its own instructions from a remote repository—what command performs this reversal?”  
**Hint**: Opposite of pushing configs, it pulls them from a central source.

---

### 13. Templating with Jinja2
**Puzzle**: “To dynamically weave your variables into config files, ye must rely on a mustache-like system. What be that system called?”  
**Hint**: An Ansible templating engine allowing placeholder replacement.

---

### 14. Terraform Remote Backend
**Puzzle**: “Store yer state on distant shores—how do ye configure Terraform to keep the ledger in S3 or Consul?”  
**Hint**: The place that holds the .tfstate file away from local disk.

---

### 15. Terraform Workspaces
**Puzzle**: “What separates dev from prod in the same config, letting ye set sail on multiple journeys within the same chart?”  
**Hint**: Lets you switch states without changing code, used for different environments.
